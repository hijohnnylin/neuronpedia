## ➡️ [**Try HeadVis**](/qwen3.5-0.8b/head/4/5?headFinder=true) - Qwen 3.5 0.8B

HeadVis ([Luger, Kamath, et al](https://transformer-circuits.pub/2026/headvis/index.html)) is Anthropic's tool for exploring attention heads, and it's now available for 37 models on Neuronpedia, for a total of 36,000+ attention head dashboards.

Let's examine an [induction head](https://www.neuronpedia.org/qwen3.5-0.8b/head/4/5?headFinder=true) (an attention head responsible for repeating previously seen token patterns like [A][B]...[A]->[B]) in [Qwen3.5-0.8B](https://www.neuronpedia.org/qwen3.5-0.8b/head/4/5?headFinder=true):

<img src="ASSET_BASE_URL/blog/headvis.gif" alt="Induction head in Qwen 3.5 0.8B" />

In the sequence above, the species name "Glenea anticepunctata" is repeated multiple times. As we hover over the tokens in its name, the orange lines show the attention head looking back at previous copies of the species name (specifically, the token immediately after the previous instance of the current token) so it knows what to output next. The grey lines show the opposite: where the hovered token is being referenced by a future token.

The rest of the attention dashboard shows other metrics and visualizations such as self attention score, max attenetion distribution, and top query and key tokens.

### Head Finder

This Qwen 3.5 model has 24 layers and 8 heads per layer, for a total of 192 attention heads, so how did we find an induction head? The **Head Finder** (enabled by clicking "Finder" on an attention head dashboard) lets us find top N heads by notable, precomputed metrics. In this case, we filtered by top **induction scores** of all heads - calculated by averaging the induction-pattern attention values across many sequences.

#### Accessing & Sharing Attention Heads

Since HeadVis is a core Neuronpedia integration, there are a few ways to access and share attention head dashboards:

1. **Model Page** - There's now an "Attention Visualizer" panel on all model pages, which contains the whole HeadVis interface and finder. Model pages on Neuronpedia are in the URL format: neuronpedia.org/[modelId], like [neuronpedia.org/qwen3.5-0.8b](https://www.neuronpedia.org/qwen3.5-0.8b).
2. **Dropdowns** - From any dashboard page or "jump to" panel on Neuronpedia, choose your model and select the "Attention Heads" release.
3. **Direct Links** - Like feature dashboards, attention heads are directly shareable simply by copying its URL, which is in the format neuronpedia.org/[modelId]/head/[layer]/[head_index], like [neuronpedia.org/qwen3.5-0.8b/head/4/5](https://www.neuronpedia.org/qwen3.5-0.8b/head/4/5). Also like feature dashbarods, you can embed attention heads in an iframe with the embed=true query parameter.
4. **Exports** - Attention head metrics and sequences based on the HeadVis [specification](https://github.com/anthropics/headvis) are downloadable in our [exports](https://neuronpedia-datasets.s3.us-east-1.amazonaws.com/index.html?prefix=v1/) under [model]/headvis/[dataset_used]. For example, the HeadVis data for Gemma-3-27B-IT is available [here](https://neuronpedia-datasets.s3.us-east-1.amazonaws.com/index.html?prefix=v1/gemma-3-27b-it/headvis/). We used pile-uncopyrighted to generate all HeadVis data.

## Natural Language Autoencoders - Community Contributions

#### Explaining Features with Foreign NLAs (Francesco Zaffino)

[Read Post (LessWrong)](https://www.lesswrong.com/posts/AtbZQuAn2iY2jCup2/explaining-sae-features-with-foreign-natural-language) | [Notebook](https://colab.research.google.com/drive/1VM-rDr9MleWzzwiGzr5rAnA8NiR33aKI#scrollTo=Hnf_egPFGrPu)

Contributor Francesco Zaffino previously demoed ([notebook](https://colab.research.google.com/drive/1DYXHO61ns2iDmiySYrp9VRFzYKMYItxI?usp=sharing)) using Gemma's NLA to explain SAE features. His new post extends this experiment in two ways:

- Cross-model NLA AV Explanations: Using one model's NLA to explain a different model's SAE features by first mapping activations from one model to another.
- Improving NLA SAE Explanations: By making the SAE vector look more like a residual stream activation (via "washout"), the explanations tend to be less influenced by random model quirks.

Check out [the post](https://www.lesswrong.com/posts/AtbZQuAn2iY2jCup2/explaining-sae-features-with-foreign-natural-language) and the associated [notebook](https://colab.research.google.com/drive/1VM-rDr9MleWzzwiGzr5rAnA8NiR33aKI#scrollTo=Hnf_egPFGrPu).

#### NLA for Gemma 4 E2B (Caleb DeLeeuw)

Contributor Caleb DeLeeuw is working on NLAs for Gemma 4 E2B, and has trained a few versions of it. Surprisingly, the NLAs were trained on a 4GB consumer GPU! These NLAs are still a work in progress, but there are two versions for experimentation:

- v0.0.1: Gemma 4 E2B [AV](https://huggingface.co/Solshine/gemma-4-e2b-nla-L23-av-v0_0_1) and [AR](https://huggingface.co/Solshine/gemma-4-e2b-nla-L23-ar-v0_0_1)
- v0.1: Gemma 4 E2B [AV](https://huggingface.co/Solshine/gemma-4-e2b-nla-L23-av-v0_1_dd-step_250) and [AR](https://huggingface.co/Solshine/gemma-4-e2b-nla-L23-ar-v0_1-paraphrase-invariant)

Example code for running these NLAs is [available here](https://github.com/SolshineCode/nla-gemma-4-e2b/blob/main/examples/round_trip_example.py).
