# jlens — Jacobian lens fitting for Neuronpedia

Fits a [Jacobian lens](https://github.com/anthropics/jlens) for the models
Neuronpedia serves and writes the results into the shared `exports/` tree.

> Based on the **jlens** reference implementation by **Anthropic PBC** —
> companion code for the *Verbalizable Workspace* paper
> (<https://github.com/anthropics/jlens>), Apache-2.0. The `jlens/` package here
> is a copy of that library; `fit_lens.py` is a Neuronpedia-adapted copy of its
> `demo/fit_lens.py` (visualisation removed, multi-GPU loading added).

## Layout

| Path | Purpose |
| --- | --- |
| `jlens/` | The lens library (copied verbatim from upstream). |
| `fit_lens.py` | Fit one model on one dataset. Streams a corpus, fits `J̄`, writes the lens + a per-prompt convergence curve. |
| `run-all-fit-lens.py` | Fit **every** model in `np_model_to_hf.json`, into the Neuronpedia `exports/` layout with a `config.yaml` per run. |
| `../np_model_to_hf.json` | Neuronpedia model id → Hugging Face model id. Shared with `headvis/` (lives in the parent `neuronpedia_utils/` dir). |

## Setup (uv)

```bash
# From this directory.
UV_HTTP_CONCURRENCY=4 UV_CONCURRENT_DOWNLOADS=4 uv sync
```

This installs torch (CUDA 12.8), transformers, accelerate, and datasets. A CUDA
GPU is required for fitting.

## Fit all Neuronpedia models

```bash
uv run python run-all-fit-lens.py \
  --dataset-name Salesforce/wikitext \
  --n-prompts 1000 \
  --stop-at-delta 2e-3 \
  --dim-batch 64
```

Each model is written to:

```
exports/<np_model_id>/jlens/<dataset>/
  <model>_jacobian_lens.pt    # the fitted lens
  <model>_convergence.csv     # per-prompt Δmean curve
  config.yaml                 # exact config, GPUs, command, attribution
  <model>_checkpoint.pt       # resumable mid-fit checkpoint (deleted on success)
```

Runs that already have a `config.yaml` + a `*_jacobian_lens.pt` are skipped, so
the command is safe to re-run. By default each model is fit on a single GPU
(`--device-map cuda`); pass `--device-map auto` to shard a model too large for
one card across all visible GPUs. Use `--dry-run` to print the per-model
commands without fitting.

**Disk usage:** downloaded weights are confined to `--hf-cache-dir`
(`/tmp/jlens-hf-cache`) and deleted after each model so the cache doesn't grow
across the whole model list. Pass `--keep-hf-cache` to retain them.

### Choosing how many prompts

The fit reports `Δmean` — the relative change of the running-mean Jacobian per
prompt — which decays like `~1/n`. By default the run stops once the smoothed
`Δmean` holds below `--stop-at-delta` (`2e-3`, ≈ where the lens has converged for
interpretability purposes), but never before `--min-prompts` (`100`), with
`--n-prompts` as the upper bound. The full curve is in each run's
`*_convergence.csv`. Tighten to `1e-3` for a bit more polish (≈2× the prompts for
marginal gain); loosen to `5e-3` for a quick, slightly noisier lens.

## Fit a single model

```bash
uv run python fit_lens.py Qwen/Qwen3.5-0.8B --out_dir out/ \
  --n_prompts 1000 --stop_at_delta 2e-3
```

See `uv run python fit_lens.py --help` for all flags (dataset selection, dtype,
`--device_map`, convergence thresholds, etc.).

## config.yaml

Each run records everything needed to reproduce the lens — the dataset and fit
hyperparameters (including defaults), the GPUs and their available VRAM at fit
time, the exact command used, results (prompts actually fitted, final `Δmean`),
and attribution. Example header:

```yaml
# Jacobian lens fit — generated by Neuronpedia run-all-fit-lens.py
# Jacobian lens ('jlens') by Anthropic PBC — ... Apache-2.0. ...
#
# GPU VRAM available when this lens was fit:
#   GPU 0 (NVIDIA H100 80GB HBM3): 79.1 GB free / 79.6 GB total
#
# Exact command used:
#   .../python fit_lens.py Qwen/Qwen3.5-0.8B --out_dir ... --n_prompts 1000 ...
#
# Generated: 2026-06-11T22:00:00+00:00
```
